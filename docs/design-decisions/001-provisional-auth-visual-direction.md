# Design Decision Record

## Decision

Provisional visual direction for the first redesign flow: Auth and Onboarding.

## Date

2026-07-17

## Status

Proposed

## Context

The project is ready to move from flow/state planning into the first real design pass. The first flow to design is Auth and Onboarding. A provisional visual direction is needed so design work can start without prematurely locking the entire website style.

## Options Considered

| Option | Pros | Cons | Risks |
|---|---|---|---|
| Formal institutional portal | Strong credibility and seriousness | Can feel dated, heavy, and bureaucratic | May repeat the current website's stiffness |
| Consumer-tech modern | Feels current, light, and approachable | Can feel too generic or startup-like | May weaken academic/institutional trust |
| Academic-modern hybrid | Balances trust, clarity, and contemporary UX | Requires careful design restraint | Needs stronger design discipline to avoid becoming visually bland |

## Decision Made

Use an academic-modern hybrid direction for Auth and Onboarding.

## Reasoning

The first user impression needs institutional trust, but the current website already suffers from outdated, confusing portal-like behavior. A hybrid direction lets the redesign feel official and grounded while still reducing friction and improving clarity.

This direction should prioritize:

- clear hierarchy
- calm spacing
- obvious next actions
- restrained but modern UI surfaces
- strong support for validation and success states

## Consequences

- Positive:
  - Supports trust without repeating the old portal feeling
  - Works well for both desktop and mobile auth flows
  - Can scale into finance/payment and credit transfer later

- Negative:
  - Requires intentional visual editing to avoid looking too generic
  - The final palette and typography still need confirmation

- Follow-up needed:
  - Convert this direction into a flow-specific design brief
  - Later confirm site-wide visual language before public discovery screens are finalized
