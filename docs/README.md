## Folders

- `flow-maps/`: Current and future flow maps.
- `state-models/`: Future-state behavioral models before screen design.
- `design-decisions/`: Decision records for important product, UX, UI, IA, and implementation choices.
- `handoff/`: Agency/developer handoff notes.
- `references/`: Reference websites, source notes, stakeholder notes, and supporting material.
- `screenshots/`: Captured screenshots from the existing website and later redesign checkpoints.
- `templates/`: Reusable templates for audit, flow mapping, and decision records.

## Documentation Standard

Use English for structured project artifacts unless Thai is explicitly requested.

Each audit or planning document should clearly distinguish:

- Observed fact
- Assumption
- UX risk
- Recommendation
- Open question
