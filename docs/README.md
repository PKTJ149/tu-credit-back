# Documentation Index

This folder contains the working documentation for the CREDIT BANK website redesign project.

## Core Documents

- [00-redesign-operating-plan.md](00-redesign-operating-plan.md): Required project sequence and working rules.
- [01-current-website-audit.md](01-current-website-audit.md): Current website inventory and first-pass audit notes.
- [02-current-page-inventory.md](02-current-page-inventory.md): Current page-by-page inventory from the source website.
- [03-current-flow-map.md](03-current-flow-map.md): Current observed flow map and cross-flow dependencies.
- [04-ux-ui-audit-findings.md](04-ux-ui-audit-findings.md): First-pass UX/UI findings with severity, impact, and recommendations.
- [05-redesign-priority-backlog.md](05-redesign-priority-backlog.md): Recommended redesign execution order after the audit.

## Folders

- `flow-maps/`: Current and future flow maps.
- `design-decisions/`: Decision records for important product, UX, UI, IA, and implementation choices.
- `handoff/`: Agency/developer handoff notes.
- `references/`: Reference websites, source notes, stakeholder notes, and supporting material.
- `screenshots/`: Captured screenshots from the existing website and later redesign checkpoints.
- `templates/`: Reusable templates for audit, flow mapping, and decision records.

## Documentation Standard

Use English for structured project artifacts unless Thai is explicitly requested.

Each audit or planning document should clearly distinguish:

- Observed fact
- Assumption
- UX risk
- Recommendation
- Open question
