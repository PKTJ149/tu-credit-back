# Public Discovery Wireframe Blueprint v1

## Purpose

This document converts the public discovery design brief and screen spec into low-fidelity structural blueprints.

It is intended to guide:

- wireframe creation
- early Figma layout
- screen composition reviews

## Wireframe Principles

- Explain before asking for commitment
- Keep browsing open before auth
- Make trust visible early
- Make programs and subjects easy to scan
- Route naturally into register/sign-in

## 1. Home

### Desktop Structure

```text
+--------------------------------------------------------------+
| Public Header                                                |
+--------------------------------------------------------------+
|  Hero / Value Proposition                                    |
|  H1 + short explanation + browse/auth CTAs                   |
+--------------------------------------------------------------+
|  Featured Programs                                           |
|  [ Program cards ]                                           |
+--------------------------------------------------------------+
|  Featured Subjects                                           |
|  [ Subject cards ]                                           |
+--------------------------------------------------------------+
|  Institutional Trust Section                                 |
+--------------------------------------------------------------+
```

## Review Checklist

- Does Home explain CREDIT BANK in one glance?
- Can users browse before auth without confusion?
- Are trust signals visible early enough?
- Are Programs and Subjects easy to compare and scan?
- Do detail pages clearly route into register/sign in when appropriate?
