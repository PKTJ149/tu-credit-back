# Learning and Registration Wireframe Blueprint v1

## Purpose

This document converts the learning/registration design brief and screen spec into low-fidelity structural blueprints.

It is intended to guide:

- wireframe creation
- early Figma layout
- screen composition reviews

## Wireframe Principles

- Planning first
- Commitment second
- History separate from active work
- Finance enters only when needed
- Active registration and academic progress should not compete visually

## 1. Learning Goals

### Desktop Structure

```text
+--------------------------------------------------------------+
| Member Shell                                                 |
+--------------------------------------------------------------+
| H1: Learning goals                                           |
| Short orientation                                             |
+--------------------------------------------------------------+
|  Top Summary Row                                             |
|  [ Goal summary ] [ Progress snapshot ] [ Recommended path ] |
+--------------------------------------------------------------+
|  Recommended Programs                                        |
|  [ Program cards ]                                           |
+--------------------------------------------------------------+
|  Recommended Subjects                                        |
|  [ Subject cards ]                                           |
+--------------------------------------------------------------+
```

### Mobile Structure

```text
+-------------------------------+
| Member Shell                  |
+-------------------------------+
| H1: Learning goals            |
| Goal summary                  |
| Progress snapshot             |
| Recommended programs          |
| [Program card]                |
| Recommended subjects          |
| [Subject card]                |
+-------------------------------+
```

## Review Checklist

- Is planning visually separate from commitment?
- Is active registration visibly different from historical progress?
- Does finance only appear when it is relevant?
- Are recommendation surfaces helpful without becoming marketing-like?
