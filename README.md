# CREDIT BANK Website Redesign

This repository is the working space for the CREDIT BANK website redesign project.

The project starts with discovery, audit, flow mapping, and redesign planning before any visual redesign or frontend implementation begins.

## Project Goal

Redesign the existing CREDIT BANK website into a clearer, more modern, more trustworthy, and more usable experience across all important user flows.

The redesign must improve both:

- User experience: navigation, content clarity, task completion, forms, payment, registration, and credit transfer.
- Product structure: sitemap, flows, reusable UI components, and future implementation readiness.

## Current Source Website

- https://creditbank.mcu.ac.th/finance.html

## Working Sequence

The project follows this required sequence:

1. Project setup
2. Current website inventory
3. Current flow mapping
4. UX/UI audit
5. Future sitemap and flow plan
6. UI component direction
7. Redesign one flow at a time
8. Handoff and validation

See [docs/00-redesign-operating-plan.md](docs/00-redesign-operating-plan.md) for the full operating plan.

## Working Rules

- Do not start visual redesign before current-state audit and flow mapping are clear.
- Design by complete user flow, not by isolated screen.
- Keep every major decision documented.
- Separate observation, assumption, recommendation, and open question.
- Prioritize user experience, simplicity, speed, and maintainability.
